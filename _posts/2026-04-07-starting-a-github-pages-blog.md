GitHub Pages와 Jekyll 조합은 개인 블로그를 빠르게 공개하고, 유지 비용을 낮게 가져가기 좋은 선택입니다. 저장소에 글을 커밋하면 배포까지 자연스럽게 이어지기 때문에 글 쓰는 흐름이 끊기지 않습니다.

## 왜 이 조합이 좋은가

- 정적 사이트라 속도가 빠릅니다.
- Git으로 글과 구조를 함께 관리할 수 있습니다.
- 테마에 의존하지 않아도 레이아웃을 직접 제어하기 쉽습니다.

## 기본 흐름

1. 저장소를 만들고 Jekyll 파일 구조를 준비합니다.
2. `_posts` 아래에 날짜 형식의 마크다운 글을 추가합니다.
3. GitHub Pages를 활성화하고 배포 소스를 지정합니다.

## 이 블로그에 적용한 방향

이번 블로그는 기본 테마를 그대로 쓰지 않고, 첫 화면에서 블로그의 성격이 분명히 보이도록 홈 레이아웃을 다시 설계했습니다. 글 목록만 나열하는 대신, 최근 글과 블로그의 편집 방향을 함께 보여주도록 구성했습니다.

```bash
bundle install
bundle exec jekyll serve
```

작게 시작해도 괜찮습니다. 중요한 건 글이 잘 보이는 구조와, 계속 업데이트할 수 있는 흐름입니다.
